# vitest-bats API Reference

## Public Exports (`vitest-bats`)

The main entry point (`package/src/index.ts`) re-exports the public API:


## Secondary Entry Points

### `vitest-bats/runtime` (`src/runtime.ts`)

Imported by virtual `.sh` modules generated by the Vite transform. Exposes
`ScriptBuilder`, `BatsResult`, and `resetAllBuilders`.

### `vitest-bats/setup` (`src/setup.ts`)

Auto-injected by `BatsPlugin` into `test.setupFiles`. Calls
`expect.extend(batsMatchers)` and registers a
`beforeEach(resetAllBuilders)` hook. Users do not import this directly;
the plugin handles it.

> The previous `vitest-bats/runner` entry point has been **removed**.
> Execution now happens inline inside `ScriptBuilder.run/exec`.

## BatsPlugin

Primary integration point. Add to the `plugins` array in `vitest.config.ts`.

```typescript
import { defineConfig } from "vitest/config";
import { BatsPlugin } from "vitest-bats";

export default defineConfig({
  plugins: [BatsPlugin()],
  test: {
    include: ["__test__/**/*.test.ts"],
    coverage: {
      enabled: true,
      provider: "v8",
    },
  },
});
```

### BatsPluginOptions

```typescript
interface BatsPluginOptions {
  /**
   * Shell script coverage mode. Default: "auto"
   * - "auto": include .sh in coverage when kcov is available and reliable
   * - true: require kcov + non-macOS, throw if unavailable
   * - false: always exclude .sh from coverage
   */
  coverage?: "auto" | boolean;

  /**
   * How to handle missing system dependencies. Default: "error"
   * - "error": throw on missing required deps, blocking all tests
   * - "warn": log warnings but continue; integration tests fail
   *   individually at runtime; pure-TS unit tests still run
   */
  deps?: "error" | "warn";

  /** Log level for detection output. Default: "errors-only" */
  logLevel?: LogLevel; // "verbose" | "debug" | "errors-only"

  /** Hyperlink format for reporter. Default: "auto" */
  links?: LinkFormat; // "auto" | "default" | "hte"
}
```

### What BatsPlugin Does

**`config` hook (Vitest config phase):**

1. Appends `"vitest-bats/setup"` to `test.setupFiles` (handles undefined,
   string, and array forms). The setup module registers matchers and a
   `beforeEach(resetAllBuilders)` hook.

**`resolveId` / `load` hooks (Vite transform phase):**

1. `resolveId(source, importer)` -- Intercepts `.sh` imports. Resolves the
   path relative to the importer. If the file exists, returns a virtual
   module ID: `\0bats:<absolute-path>`.
2. `load(id)` -- For virtual IDs, generates JavaScript that imports
   `ScriptBuilder` from `vitest-bats/runtime` and exports a fresh instance
   bound to the script path.
3. Registers each script path in `registeredScripts` and writes a
   `scripts.json` manifest to `<cacheDir>/vitest-bats/`.

**`configureVitest` hook:**

1. Resolves cache directory from Vitest's `cacheDir` (falling back to
   Vite's `cacheDir`, finally `node_modules/.vitest`), scoped under
   `vitest-bats/`.
2. Checks system dependencies and validates BATS version >= 1.5
   (required for `run --separate-stderr`). Throws on missing required
   dependencies (with install instructions) when `deps: "error"`; logs
   warnings when `deps: "warn"`.
3. Sets environment variables: `BATS_PATH`, `BATS_SUPPORT_PATH`,
   `BATS_ASSERT_PATH`, `BATS_MOCK_PATH`, `KCOV_PATH`, `KCOV_LOG_LEVEL`.
4. Resolves coverage mode. Always exports `__VITEST_BATS_CACHE_DIR__`.
   When kcov is reliable and shell coverage enabled, sets
   `__VITEST_BATS_KCOV__=1`.
5. Always injects `BatsCoverageReporter` when Vitest coverage is enabled.
   Passes `CoverageThresholds` from the config and `statementPassThrough =
   true` when kcov is unreliable.

### Coverage Mode Resolution

| `coverage` option | kcov available | macOS? | Result |
| --- | --- | --- | --- |
| `"auto"` (default) | Yes | No | Shell coverage ON |
| `"auto"` | Yes | Yes | Shell coverage OFF (SIP) |
| `"auto"` | No | -- | Shell coverage OFF |
| `true` | Yes | No | Shell coverage ON |
| `true` | Yes | Yes | Throws Error |
| `true` | No | -- | Throws Error |
| `false` | -- | -- | Shell coverage OFF |

## ScriptBuilder

Config-then-execute builder returned by `.sh` imports. Accumulates state
(`env`, `flags`, `mock`) and consumes it on `run()` / `exec()` to produce
a `BatsResult`. Every call to `run()` or `exec()` resets the builder's
state via a `finally` block.

### Usage Pattern

```typescript
import { describe, expect, test } from "vitest";
import hello from "../scripts/hello.sh";

describe("hello.sh", () => {
  test("outputs default greeting", async () => {
    const result = await hello.run();
    expect(result).toSucceed();
    expect(result).toContainOutput("Hello World");
  });

  test("greets by name", async () => {
    const result = await hello.run("--name", "Alice");
    expect(result).toSucceed();
    expect(result).toHaveJsonValue("greeting", "Hello Alice");
  });
});
```

`await` is optional but idiomatic — `BatsResult` is not thenable, so
`await batsResult` resolves to the same instance per the JS Promise spec.

### Properties

| Property | Type | Description |
| --- | --- | --- |
| `path` | `string` (readonly) | Absolute path to the shell script |
| `name` | `string` (readonly) | Script basename (e.g., `"hello.sh"`) |

### Configuration Methods

All configuration methods return `this` for chaining and accumulate state
that is consumed by the next `run()` / `exec()` call.

#### `env(vars: Record<string, string>): this`

Set environment variables for the next run. Variables accumulate across
multiple `env()` calls until consumed.

```typescript
const result = await hello.env({ NAME: "Alice", DEBUG: "1" }).run();
```

Generates: `NAME="Alice" DEBUG="1" run --separate-stderr "$SCRIPT"`

#### `flags(value: string): this`

Set flags appended to the script invocation. Replaces any prior `flags()`
call (does not accumulate).

```typescript
const result = await hello.flags("--json").run();
```

Generates: `run --separate-stderr "$SCRIPT" --json`

#### `mock(cmd: string, responses?: Record<string, string>): this`

Register a recorder shim for `cmd`. Each call to `mock()` accumulates a
new shim. The optional `responses` map keys patterns (e.g., `"remote
get-url *"`) to bash response expressions (e.g.,
`"echo https://example.com"`). Empty `responses` records calls and exits 0.

```typescript
const result = await gitScript
  .mock("git", { "remote get-url *": "echo https://example.com" })
  .run();
```

Patterns are converted to bash globs by `patternToBashGlob()` (literal
segments are double-quoted; `* ? [ ]` stay active). Responses are
evaluated via `eval` so they can include any bash expression.

When no pattern matches at runtime, the shim emits an error to stderr and
exits 1.

### Execution Methods

#### `run(...args: string[]): BatsResult`

Generate a `.bats` file that invokes `"$SCRIPT" arg0 arg1 ...` (with
current `env`, `flags`, and `mock` state), execute it via `bats --tap`,
and return a `BatsResult`. Resets builder state in `finally`.

```typescript
const result = await hello.run();                       // "$SCRIPT"
const result = await hello.run("--name", "Alice");     // "$SCRIPT" "--name" "Alice"
```

#### `exec(shellExpr: string): BatsResult`

Generate a `.bats` file that runs `bash -c '<expr>'`. Same lifecycle as
`run`. `$SCRIPT` is exported in `setup()` so it's available inside the
expression. Useful for shell pipelines.

```typescript
const result = await hello.exec('"$SCRIPT" --name Pipeline');
const result = await hello.exec('"$SCRIPT" | cat');
```

`exec()` is **not** kcov-wrapped — the user controls what runs.

### Lifecycle Methods

#### `reset(): void`

Clear `env`, `flags`, and `stubs`. Called automatically after every
`run()` / `exec()` via `finally`, and also from `resetAllBuilders()` in
the auto-injected `beforeEach` hook.

## BatsResult

Plain class returned by `ScriptBuilder.run/exec`.

```typescript
class BatsResult {
  readonly status: number;
  readonly output: string;
  readonly stderr: string;
  readonly lines: string[];           // output.split("\n"), [] if output is ""
  readonly stderr_lines: string[];    // stderr.split("\n"), [] if stderr is ""
  readonly calls: Record<string, MockCall[]>;

  constructor(data: BatsResultData);
  json<T = unknown>(): T;             // Lazy parse with WeakMap caching
}
```

### `BatsResultData`

```typescript
interface BatsResultData {
  status: number;
  output: string;
  stderr: string;
  calls: Record<string, MockCall[]>;
}
```

### `MockCall`

```typescript
interface MockCall {
  args: string[];
}
```

### `result.json<T>()`

Lazily parses `output` as JSON, caching the parsed value on first call.
Throws `Error` with a descriptive message if `output` is not valid JSON.

```typescript
const data = result.json<{ greeting: string }>();
expect(data.greeting).toBe("Hello World");
```

## resetAllBuilders

```typescript
function resetAllBuilders(): void;
```

Iterates the module-local builder set and calls `.reset()` on each.
Called automatically from the auto-injected `beforeEach` hook in
`setup.ts`. Exported for advanced users.

## Matchers (`batsMatchers`)

Twenty-three Vitest matchers, auto-registered via `expect.extend()` by the
setup module. All matchers operate on a `BatsResult` value, identified via
the `BATS_RESULT_BRAND` symbol (registered with `Symbol.for(...)` so it's
shared across module-graph duplicates). `instanceof` would fail because
the rslib build produces self-contained per-entry bundles, each with its
own copy of the `BatsResult` class.

### Status

| Matcher | Asserts |
| --- | --- |
| `toSucceed()` | `result.status === 0` |
| `toFail(code?)` | `code === undefined ? status !== 0 : status === code` |

### Output (stdout)

| Matcher | Asserts |
| --- | --- |
| `toHaveOutput(text)` | `result.output === text` |
| `toContainOutput(text)` | `result.output.includes(text)` |
| `toMatchOutput(pattern)` | RegExp test against `result.output` |
| `toHaveEmptyOutput()` | `result.output === ""` |

### Stderr

| Matcher | Asserts |
| --- | --- |
| `toHaveStderr(text)` | `result.stderr === text` |
| `toContainStderr(text)` | `result.stderr.includes(text)` |
| `toMatchStderr(pattern)` | RegExp test against `result.stderr` |

### Lines

| Matcher | Asserts |
| --- | --- |
| `toHaveLine(index, text)` | `result.lines[index] === text` |
| `toHaveLineContaining(text, index?)` | If `index` given, that line contains `text`; else any line contains `text` |
| `toHaveLineMatching(pattern, index?)` | Same as above but RegExp |
| `toHaveLineCount(n)` | `result.lines.length === n` |

### JSON

| Matcher | Asserts |
| --- | --- |
| `toOutputJson()` | `result.json()` doesn't throw |
| `toEqualJson(expected)` | Deep equality between parsed JSON and `expected` |
| `toMatchJson(partial)` | `partial` is a deep subset of parsed JSON |
| `toHaveJsonValue(path, expected)` | Dotted/bracketed path exists and deep-equals `expected` |
| `toHaveJsonPath(path)` | Dotted/bracketed path exists |

Path syntax for `toHaveJsonValue` / `toHaveJsonPath`: dot for keys,
brackets for array indices (e.g., `"users[0].name"`).

### Schema

| Matcher | Asserts |
| --- | --- |
| `toMatchSchema(schema)` | Standard Schema OR raw JSON Schema validates `result.json()` |
| `toMatchJsonSchema(schema)` | Raw JSON Schema validates `result.json()` |

`toMatchSchema` auto-detects Standard Schema (Zod, Valibot, Arktype,
Effect Schema) by the `~standard` property. Raw JSON Schemas are compiled
via Ajv (cached in a `WeakMap`).

### Invocation

| Matcher | Asserts |
| --- | --- |
| `toHaveInvoked(cmd, opts?)` | `cmd` was invoked at least once; if `opts.args` given, at least one call matches those args |
| `toHaveInvokedTimes(cmd, n)` | `result.calls[cmd].length === n` |
| `toHaveInvokedExactly(cmd, calls)` | `result.calls[cmd]` deep-equals the given call list (order-sensitive) |

> Strict-count matchers can be flaky when kcov is enabled and the
> instrumented script invokes binaries kcov uses internally (e.g., `git
> rev-parse` during instrumentation). Mock binaries kcov never touches for
> reliable strict-count assertions — see `__test__/integration/mocks.int.test.ts`.

### Type Augmentation

`matchers.ts` augments `vitest`'s `Assertion` and
`AsymmetricMatchersContaining` interfaces via `declare module "vitest"`.
The root workspace mirrors this in `types/vitest-matchers.d.ts` because
API Extractor strips `declare module` from rolled-up `.d.ts`.

## Schema Validation

### `validate(schema, value): ValidationResult`

Single entry point used by `toMatchSchema` and `toMatchJsonSchema`.

```typescript
type ValidationResult = { ok: true } | { ok: false; issues: string[] };
```

- If `schema` has a `~standard` property, dispatches to the Standard
  Schema validator. Async validators throw with a clear migration message.
- Otherwise, treats `schema` as a raw JSON Schema object and compiles via
  Ajv. Compiled validators are cached in a `WeakMap` keyed by the schema
  object.
- Throws if `schema` is null/undefined or not an object.

### `isStandardSchema(schema): boolean`

Type guard: returns `true` if `schema` has a `~standard.validate`
function.

## generateBatsFile

Pure function that produces the `.bats` file string for a single execution.

```typescript
function generateBatsFile(input: GenerateInput): string;

interface GenerateInput {
  scriptPath: string;                 // Absolute path to the script
  args: string[];                     // Args passed to "$SCRIPT" (mode: "args")
  env: Record<string, string>;        // Env vars for the run
  flags: string;                      // Flags appended to the invocation
  stubs: StubSpec[];                  // Recorder shims to write
  recorderDir: string;                // Per-execution scratch dir
  deps: BatsDeps;                     // bats / bats-support paths
  mode: RunMode;                      // "args" or "shell" expression
  kcov?: KcovConfig;                  // Optional kcov wrapping
}

interface BatsDeps {
  batsPath: string;
  batsSupportPath: string;
}

interface StubSpec {
  cmd: string;
  responses: Record<string, string>;  // pattern -> bash response expression
}

type RunMode =
  | { kind: "args" }
  | { kind: "shell"; expression: string };

interface KcovConfig {
  kcovPath: string;
  outputDir: string;
}
```

Wraps the script invocation with kcov when `kcov` is provided and `mode`
is `"args"`. Shell mode is never kcov-wrapped (the user controls what
runs).

## executeBats / parseExecutionResult

```typescript
interface ExecuteOptions {
  batsPath: string;
  batsContent: string;
  recorderDir: string;
  timeoutMs?: number;                 // Default 60000
}

function executeBats(opts: ExecuteOptions): BatsResultData;
function parseExecutionResult(recorderDir: string): BatsResultData;
```

`executeBats` writes `batsContent` to `os.tmpdir()/vitest-bats-<pid>/`,
spawns `bats --tap <file>` synchronously, then calls
`parseExecutionResult` to read `result.txt` (status + base64
output/stderr) and `calls.jsonl` (mock invocations) from `recorderDir`.

A non-zero BATS exit code is **not** treated as a tooling error — the
per-test status comes from `result.txt`. Only a missing `result.txt`
throws.

The temp directory is registered for cleanup via `process.on("exit")`
(best-effort).

## BatsCoverageReporter

Merges shell script coverage into Vitest's v8 Istanbul CoverageMap.
Always injected by `BatsPlugin` when Vitest coverage is enabled.

### Constructor

```typescript
constructor(cacheDir: string, options?: {
  thresholds?: CoverageThresholds;
  statementPassThrough?: boolean;
})
```

### CoverageThresholds

```typescript
interface CoverageThresholds {
  statements: number;
  branches: number;
  functions: number;
  lines: number;
}
```

Read from `vitest.config.ts` `coverage.thresholds` by the plugin (with `??
0` defaults for missing dimensions).

### Hooks

#### `onInit(): void`

Clears `<cacheDir>/kcov` at the start of each Vitest session so coverage
merge only sees data from the current run.

#### `onCoverage(coverage: unknown): void`

Three-path strategy per script in `scripts.json`:

1. **Real kcov data** -- Walks
   `<cacheDir>/kcov/<id>/<scriptName>/cobertura.xml` two levels deep.
   Multiple files per script merged via `Math.max` per statement.
   Synthetic branch/function entries at threshold level always applied
   (kcov tracks only statements/lines for bash).
2. **Statement pass-through** -- When `statementPassThrough` is true and
   thresholds are configured, synthesizes all four coverage dimensions at
   the exact threshold percentage. Prints a one-line warning per session
   listing affected scripts.
3. **Zero-coverage fallback** -- Builds entries from source line analysis
   (skipping comments, structural keywords, bare braces) with all hits
   = 0.

### Synthetic Coverage Algorithm

`thresholdFraction(threshold)` uses GCD reduction to find the smallest
total/covered pair (e.g., 50% -> 1/2, 75% -> 3/4, 80% -> 4/5). Statement
counts are padded to be divisible by the denominator for exact
percentages. Synthetic branches use a single multi-arm branch; synthetic
functions create the minimal number of entries.

### Cobertura XML Parsing

Uses `fast-xml-parser` (`ignoreAttributes: false`,
`attributeNamePrefix: "@_"`). Extracts source path, package/class/line
data, line numbers (`@_number`), and hit counts (`@_hits`). Converts to
Istanbul format with `statementMap` and `s` records. `fnMap` / `branchMap`
/ `f` / `b` start empty and are populated by `applySyntheticBranches` /
`applySyntheticFunctions` when thresholds are configured.

## System Dependencies

| Dependency | Required | Detected | Notes |
| --- | --- | --- | --- |
| bats | Always (>= 1.5) | `command -v bats` + `bats --version` parse | `run --separate-stderr` requires 1.5+ |
| bats-support | Always | Library path search | Loaded by generated `setup()` |
| jq | Always | `command -v jq` | Used by recorder shims to write `calls.jsonl` |
| kcov | Coverage only | `command -v kcov` | Blocked by SIP on macOS |

`bats-assert` and `bats-mock` are NOT detected. Assertions happen JS-side
(via `expect.extend` matchers) and the recorder shims are self-contained,
so neither library is needed at runtime.
